HTML, CSS, DOM, pt.1

10 mar 18

Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

HTML, to me, is the skeleton of the body you are building: it defines what amount of and where the limbs, brain, and vital organs are.
CSS, is used to define how the body appears; what colour, what length, etc.
DOM, is commonly referred to as a "tree" with multiple branches or layers. The DOM allows us to interact, modify, and remove elements -- or in this analogy, limbs -- of markup (or body).

What is meant by boxifying design?

Given that all HTML elements can be considered as boxes, I think it means to look at elements of design as a series, or array of boxes. The content of the design is contained within each of these boxes, so it pays to dissect each section of a design into boxes; and build / form these boxes with your HTML.

What is the box model?

As above, All HTML elements can be considered as boxes. In CSS, the term "box model" is used when talking about design and layout. The CSS box model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content.
